# Recommended Foundation Gate

Do not create the new clean Magna project yet. Approve foundation creation only when all gates below have explicit evidence and Vinay’s recorded decision.

1. **Authority gate:** stage/repository decisions receive new accepted IDs; contrary decisions are marked `SUPERSEDED`, not rewritten or deleted.
2. **Canonical truth gate:** Magna/HELIX/Cosmos/Identity and capability status are reconciled; BRS disposition and SGN block are explicit.
3. **Architecture gate:** reusable components are selected through the policy integration experiment; contracts and non-goals are frozen.
4. **TRACE dual-plane gate:** runtime-to-engineering schema, independent verifier, privacy boundaries, and anti-self-certification rule are approved.
5. **UX gate:** desktop information architecture, task-history behavior, approval/recovery states, accessibility target, responsive boundary, and performance budget are accepted.
6. **Environment gate:** dev/test/integration/staging/UAT/production/DR responsibilities, secrets, deployment, rollback, backup, and observability are documented and testable.
7. **Backlog gate:** one approved backlog has requirements, acceptance criteria, DoR/DoD, dependencies, risks, and requirement→code→test→evidence identifiers.
8. **Validation gate:** Command Center baseline remains green; Enso pytest collection is corrected and independently reviewed; TRACE frontend is reproducibly buildable; raw evidence format is standardized.
9. **Governance gate:** repository ownership, branch protection, commit/release authority, dependency policy, and cloud-data consent are explicit.
10. **Creation decision:** Vinay explicitly authorizes repository creation, initial scope, and first sprint. Passing earlier gates does not imply authorization.

Recommended independent reviewer: a senior security architect/QA lead with no authorship in Command Center, Enso Sprint 5, or this report, tasked to reproduce validations and challenge policy bypass, evidence provenance, and environment/release assumptions.
