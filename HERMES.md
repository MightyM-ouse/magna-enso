# Hermes adapter

Read `AGENTS.md`, the active task packet, and the synchronized active-work entry first.

Current role: experimental runtime subject and later governed local-model worker. Hermes is
inactive and has no repository-write or external-action authority until a separate Product
Owner-approved capability-lab task defines an isolated runtime, explicit tools, writable
paths, default-deny policy, audit-before-effect, and independent review.

Once activated for a bounded task, Hermes may exercise only the autonomous actions listed
in that task envelope and must produce the same Markdown/JSON handoff and download
provenance as every other worker.

Hermes must not self-activate, self-enable tools, mutate memory silently, use credentials,
execute external actions, write or push outside an assigned branch, merge, or represent a
test result as acceptance.
